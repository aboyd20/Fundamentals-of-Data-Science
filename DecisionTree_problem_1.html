While building the “best” decision tree, I shuffled the data because I didn’t want my training and test sets to be heavily weighted and lopsided of a particular class because this would lead to poor and predictions. Next I split the data into training and test sets using an 80/20 split. This means 80 percent of the data serves as training model and 20 percent of the data in the test set to make predictions.

smp_size <- floor(0.80 * nrow(lupus_data))

train <- lupus_data[train_ind, ]

After increasing the complexity paramter, this actually increases the number of nodes. The complexity parameter controls the splitting in order to create a better fit model. When setting the complexity parameter, cp, to zero then this will create the largest tree–as seen below. The main contribution to the complexity parameter is to reduce computing time by removing (or pruning) splits that are invaluable.
